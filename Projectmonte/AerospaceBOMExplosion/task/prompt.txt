You are an aerospace manufacturing analyst. Using the input files in this directory, produce five output CSV files. Analysis date: 2024-06-01. Round monetary values to 4 decimal places; quantities to 4 decimal places unless stated.

Input files: assemblies.csv (demand_quantity, priority, due_date), components.csv (category, unit_cost, default_supplier, lead_time_days, min_order_qty, is_critical), bom_structure.csv (parent_id, child_id, quantity_per, level, scrap_rate; parents may be ASM/SUB/MOD), inventory.csv (on_hand_qty, allocated_qty, on_order_qty), substitutions.csv (approval_status, conversion_factor), supplier_pricing.csv (tiered min_qty/max_qty/unit_price), labor_rates.csv (labor_rate_per_hour, run_hours_per_unit, overhead_rate per category), expedite_fees.csv (expedite_pct, max_expedite_fee per supplier), supplier_contracts.json (nested supplier_id → category → [{effective_from, effective_to, discount_rate}]; not all pairs present), production_schedule.xlsx (read sheet "operational_schedule" only; columns month YYYY-MM, priority_scope, capacity_multiplier), delivery_performance.parquet (supplier_id, scheduled_date, status ON_TIME/LATE in chronological order).

OUTPUT 1 — supplier_scorecard.csv
Per supplier from delivery_performance: total_deliveries, on_time_deliveries, reliability_score (on_time/total, 4 dp), consecutive_late_count (trailing LATE count at the end of the sequence — inclusive; 0 if the last delivery is ON_TIME), average_delay_days (mean delay in calendar days for LATE rows only, 2 dp; 0.0 if none). Sort by supplier_id.

OUTPUT 2 — exploded_requirements.csv
Recursively explode each assembly's BOM: gross_qty = parent_demand × quantity_per × (1 + scrap_rate), recurse into SUB/MOD until reaching leaf components, accumulate per (assembly_id, component_id). Columns: demand_qty, gross_qty_required, net_qty_required (gross − max(0, on_hand − allocated), floor 0), unit_price (tiered at gross_qty_required), contract_discount, discounted_unit_price. For contract_discount: look up supplier_contracts.json by default_supplier then category. If the key pair is absent → 0.0. If present but no entry satisfies effective_from ≤ 2024-06-01 ≤ effective_to → 0.0. Otherwise use that entry's discount_rate. discounted_unit_price = unit_price × (1 − contract_discount). Sort by assembly_id, component_id.

OUTPUT 3 — material_shortages.csv
Rows where net_qty_required > 0. For each: available_substitute_qty = APPROVED substitute's on_hand (unreduced). shortage_covered = YES if available_substitute_qty ≥ net_qty_required, else NO. shortage_severity = CRITICAL if is_critical = YES or net_qty_required > 50, else STANDARD. Sort by assembly_id, component_id.

OUTPUT 4 — purchase_orders.csv
One PO row per shortage where shortage_covered = NO. qty_ordered = ceiling(max(net_qty_required, min_order_qty)). unit_price = tiered price at qty_ordered. po_value = qty_ordered × unit_price. expedite_fee = min(po_value × expedite_pct, max_expedite_fee) from expedite_fees.csv. base_lead_time from components.csv. consecutive_late_count from supplier_scorecard for the component's default_supplier. adjusted_lead_time = ceiling(base_lead_time × (1 + 0.1 × consecutive_late_count)). reliability_adjusted = YES if consecutive_late_count > 0, else NO. Sort by assembly_id, component_id.

OUTPUT 5 — cost_rollup.csv
Per assembly: capacity_multiplier from sheet "operational_schedule" — match month (YYYY-MM of due_date) with priority_scope equal to the assembly's priority; if no exact match use the row with priority_scope = "ALL" for that month; if no such row exists use 1.0. material_cost = Σ gross_qty × unit_price. discounted_material_cost = Σ gross_qty × discounted_unit_price. effective_material_cost = discounted_material_cost × capacity_multiplier. labor_cost = Σ gross_qty × run_hours_per_unit × labor_rate_per_hour × overhead_rate per category. total_cost = effective_material_cost + labor_cost. Sort by assembly_id.
