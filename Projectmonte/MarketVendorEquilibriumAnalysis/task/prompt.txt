Generate market vendor equilibrium analysis for 20 vendors across 4 segments over January to June 2024 using YYYY-MM month format. Read all CSV input files from the task directory. Produce five output files in the task directory: vendor_monthly.csv (120 rows, one per vendor per month), segment_equilibrium.csv (24 rows, one per segment per month), vendor_risk_profile.csv (20 rows), promotion_effectiveness.csv (one row per promo-vendor pair with qualifying transactions, sorted by promo_id then vendor_id), and market_health_ledger.csv (only unhealthy segment-months).

For vendor_monthly.csv, per vendor per month compute: gross_revenue as sum of (quantity * unit_price * (1 - discount_pct) + shipping_fee) for COMPLETED and RETURNED transactions only, rounded to 2 decimals. CANCELLED and DISPUTED transactions are excluded entirely. Null shipping_fee means 0, null discount_pct means 0, null rating means exclude from averages. net_revenue equals gross_revenue minus the same line-item total for RETURNED transactions only, rounded to 2 decimals. commission_amount equals net_revenue times the rate from commission_rates.csv matched on vendor tier and segment where the 15th of the month falls within effective_from to effective_to inclusive, rounded to 2 decimals. seasonal_adjusted_revenue equals net_revenue times demand_factor from seasonal_factors.csv matched on vendor segment and month, rounded to 2 decimals. adjusted_return_rate equals (RETURNED count / (COMPLETED + RETURNED count)) times return_adjustment from seasonal_factors, rounded to 4 decimals. avg_rating is mean of non-null ratings across all status types, rounded to 2, or 0.0 if none. complaint_count and critical_complaints counted from complaints where complaint_date falls in the month. avg_resolution_days is mean of non-null resolution_days for that month, rounded to 2, or 0.0. weighted_shipping equals sum of shipping_fee times shipping_weight from seasonal_factors over COMPLETED and RETURNED transactions, rounded to 2 decimals.

VSI is computed sequentially per vendor month by month. First month: base_score = avg_rating * 20 (or 50 if 0), subtract penalty = critical_complaints * 5 + other_complaints * 1.5, clamp 0 to 100, round to 2 decimals. Subsequent months: if adjusted_return_rate < vendor max_return_pct, blend 0.7 * prev_VSI + 0.3 * current_base_score, round to 2. If adjusted_return_rate >= max_return_pct, multiply prev_VSI by 0.85 (if rate < 1.5 * max_return_pct) or 0.6 (otherwise), round to 2. Clamp 0 to 100 after each step.

For segment_equilibrium.csv, group vendors by segment from vendors.csv. vendor_count = vendors with positive net_revenue. total_revenue = segment sum. Compute share = vendor_net_revenue / total_revenue. hhi = sum of squared shares, rounded to 6. shannon_entropy H = -sum(share * log2(share)) for share > 0, rounded to 4. VCI: for months after first, collect shares for vendors in both current and previous month, compute variance(prev_shares) / variance(curr_shares), round to 4. If current variance is 0, VCI = 99.0. First month VCI = 1.0. concentrated_vendors = count of vendors with share > concentration_threshold from market_segments.csv. MSI = alpha * (1/(H+0.01)) + beta * VCI + gamma * (1/(concentrated_vendors+1)) with alpha, beta, gamma from market_segments.csv. Round each of the three MSI components to 4 decimals before summing, then round sum to 4.

Health classification in priority order: CRITICAL if H < entropy_floor AND VCI > vci_ceiling. DETERIORATING if H < entropy_floor OR (VCI > vci_ceiling AND MSI > 5.0). STAGNANT if MSI > 3.0 AND concentrated_vendors <= 1. TRANSITIONING if abs(VCI - 1.0) > 0.5. HEALTHY if H >= entropy_floor AND MSI <= 2.0. Otherwise STABLE. Track unhealthy_streak as consecutive non-HEALTHY non-STABLE months per segment, reset to 0 on HEALTHY or STABLE.

For vendor_risk_profile.csv: avg_vsi, min_vsi, max_vsi rounded to 2. total_complaints, critical_complaint_count from all complaints. avg_monthly_revenue = mean(net_revenue), rounded to 2. revenue_volatility = std/mean of monthly net_revenue, rounded to 4. total_return_rate = mean of monthly adjusted_return_rate, rounded to 4. consecutive_decline_months = max consecutive months where VSI dropped from previous. Risk category priority: HIGH_RISK if avg_vsi < 40 or consecutive_decline_months >= 4. MODERATE_RISK if avg_vsi < 60 or total_complaints > 30 or revenue_volatility > 0.5. WATCH if avg_vsi < 75 and critical_complaint_count > 0. Otherwise LOW_RISK. worst_month = month with lowest VSI.

For promotion_effectiveness.csv, match transactions to promotions on promo_id and vendor_id for COMPLETED transactions where transaction_date is between start_date and end_date inclusive. total_promo_revenue = sum of line revenues, rounded to 2. roi = (revenue - budget_cap) / budget_cap, rounded to 4. lift_ratio = avg promo transaction revenue / avg non-promo transaction revenue for same vendor in same date range, rounded to 4 (0 if no comparisons). avg_discount_applied = mean discount_pct, rounded to 4. effective_discount = min(avg_discount, max_discount_pct), rounded to 4.

For market_health_ledger.csv, only rows where health_class is not HEALTHY or STABLE. entropy_gap = max(0, entropy_floor - H), rounded to 4. vci_excess = max(0, VCI - vci_ceiling), rounded to 4. penalty_score = entropy_gap * 10 + vci_excess * 5, rounded to 2. decay_factor = health_decay_rate ^ unhealthy_streak, rounded to 4. period_penalty = penalty_score * decay_factor, rounded to 2. cumulative_penalty = running sum of period_penalty per segment, rounded to 2 each step, reset on HEALTHY or STABLE. credit_amount = penalty_score * 0.1 if vendor_count >= min_vendor_count, else 0, rounded to 2. net_penalty = cumulative_penalty - credit_amount, rounded to 2.
