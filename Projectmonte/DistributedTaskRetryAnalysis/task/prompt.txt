Generate distributed task retry analysis for 15 tenants, 12 pools across 4 regions (ALPHA/BETA/GAMMA/DELTA), Jan-Jun 2024. job_metrics must have 528 rows: one per job. Use YYYY-MM format for month columns. Read all CSV input files from the task directory.
Rounding policy: monetary values and durations round to 2 decimal places. Rates and ratios round to 4 decimal places. Penalty intermediate steps round to 4 decimal places before final 2-place round. Reliability scores round to 3 decimal places at each sequential step. Memory peaks round to 1 decimal place. CPU utilization rounds to 4 decimal places.
Null handling: ended_at empty for TIMEOUT attempts means the attempt ran until the job sla_seconds elapsed from started_at. cpu_ms empty means 0. mem_peak_mb empty means 0.0. error_code empty on FAIL status means UNKNOWN. error_code empty on OK/TIMEOUT/REJECT means no error.
Backoff delay per retry attempt (attempt_num > 1). Look up retry config matching tenant service_level, job priority, and attempt queued_at within effective_from to effective_to. FIXED: base_delay_sec. LINEAR: base_delay_sec × attempt_num. EXPONENTIAL: base_delay_sec × 2^(attempt_num − 2). DECORRELATED: min(max_delay_sec, base_delay_sec + previous_delay × 1.5) where previous_delay defaults to base_delay_sec for first retry. Cap all at max_delay_sec.
SLA breach: total_duration_sec = final attempt ended_at minus submitted_at. Breached when total_duration_sec exceeds sla_seconds. breach_seconds = total_duration − sla_seconds if breached, else 0.
Penalty for breached jobs: look up sla_terms by tenant service_level and job priority. effective_breach = max(0, breach_seconds − grace_sec). base_penalty = (effective_breach / 60) × penalty_per_min. escalation_factor = 1 + escalation_pct × retry_count. escalated_penalty = base_penalty × escalation_factor. If retry_count exceeds floor(max_retries / 2), apply a 15% repeated-failure surcharge to escalated_penalty. Apply service-level penalty adjustment after surcharge: PLATINUM 0.80, GOLD 0.95, SILVER 1.10, BRONZE 1.25. If the job is incident-affected, apply a severity-dependent discount based on the highest-severity incident overlapping any attempt: S1 = 60%, S2 = 40%, S3 = 20%, S4 = 10%. If breach_seconds exceeds 2× sla_seconds, multiply by critical_mult.
Incident overlap: an attempt is incident-affected if started_at < incident end_time AND ended_at > incident start_time, on the same pool. A job is incident-affected if any of its attempts are.
Outcome classification (evaluate in priority order): SUCCESS = final OK, no breach. DELAYED_SUCCESS = final OK, breached. EXHAUSTED = retry_count >= max_retries, not OK. INCIDENT_FAILURE = incident-affected, not OK. TIMEOUT_FAILURE = final TIMEOUT, not incident-affected. ERROR_FAILURE = all other non-OK.
penalty_tier classification: NONE = no breach. GRACE = breached but effective_breach is 0. STANDARD = effective_breach > 0 and breach_seconds ≤ 1.5× sla_seconds. SEVERE = breach_seconds > 1.5× sla_seconds and ≤ 2× sla_seconds. CRITICAL = breach_seconds > 2× sla_seconds.
job_metrics.csv: job_id, tenant_id, priority, total_attempts, retry_count, final_status, total_duration_sec, total_exec_sec (sum of all attempt durations), total_cpu_ms, peak_mem_mb (max across attempts), sla_breached (YES/NO), breach_seconds, expected_delay_sec (sum of computed backoff delays), actual_wait_sec (sum of positive inter-attempt gaps: max(0, queued_at − previous ended_at)), delay_deviation_sec (actual_wait − expected_delay), incident_affected (YES/NO), penalty_amount, error_summary (pipe-separated sorted unique error codes from all attempts), outcome_class, penalty_tier, penalty_weight (for breached jobs: penalty_amount × 60 / max(0.01, breach_seconds); for non-breached: 0.0).
tenant_monthly.csv: per tenant per month (month determined by attempt queued_at). tenant_id, month, service_level, total_jobs, total_attempts, total_retries (attempts with attempt_num > 1), success_rate (final OK / total_jobs), avg_duration_sec (mean total_duration_sec), retry_rate (total_retries / total_attempts), total_cpu_ms, peak_mem_mb (max), base_cost, retry_cost, penalty_cost, total_cost, sla_breaches (count), incident_jobs (count), cost_efficiency_ratio (total_cost / max(1, total_jobs)), risk_score ((1 − success_rate) × 40 + retry_rate × 30 + (sla_breaches / max(1, total_jobs)) × 30), reliability_score, health_status, trend.
Cost: cpu_hours = total_cpu_ms / 3600000. avg_cpu_cost = mean cost_per_cpu_hr across pools used that month. Rate card by tenant region + month for cpu_multiplier, mem_multiplier, retry_surcharge_pct. base_cpu_cost = cpu_hours × avg_cpu_cost × cpu_multiplier. mem_gb_hours = (sum mem_peak_mb / 1024) × (sum duration_sec / 3600). base_mem_cost = mem_gb_hours × 0.01 × mem_multiplier. base_cost = base_cpu_cost + base_mem_cost. retry_cost = base_cost × retry_surcharge_pct / 100 if any retries else 0. penalty_cost = sum of job penalties. total_cost = base_cost + retry_cost + penalty_cost.
Reliability score: per tenant-month, computed sequentially in chronological order starting at 50.0. sr_input = success_rate × 100. rr_penalty = retry_rate × 30. breach_penalty = min(20, (sla_breaches / max(1, total_jobs)) × 100). raw_score = sr_input − rr_penalty − breach_penalty. Blending: success_rate ≥ 0.90 uses 70/30 (prev/raw), 0.70–0.89 uses 50/50, below 0.70 uses raw_score only. Clamp 0–100 after each step.
trend: compare reliability_score to previous month. IMPROVING if current > previous + 2. DECLINING if current < previous − 2. STABLE otherwise. First month for each tenant uses initial score 50.0 as previous.
Health status (priority order): breach_ratio = sla_breaches / max(1, total_jobs). CRITICAL if score < 30 OR (success_rate < 0.50 AND breach_ratio > 0.40). DEGRADED if score < 50 OR (success_rate < 0.70 AND breach_ratio > 0.25). AT_RISK if score < 70 OR breach_ratio > 0.15. HEALTHY if score ≥ 85 AND success_rate ≥ 0.90 AND breach_ratio ≤ 0.05. Otherwise STABLE.
pool_monthly.csv: per pool per month (attempt queued_at). pool_id, month, region, tier, total_attempts, ok_count, fail_count, timeout_count, reject_count, total_cpu_ms, total_duration_sec, avg_duration_sec, peak_mem_mb, cpu_utilization_pct = total_cpu_ms / (cpu_units × 1000 × 3600 × 730) × 100, top_error (most frequent error_code excluding blanks; alphabetically first if tied; empty if none), incident_minutes (total duration of incidents starting in that month for that pool).
retry_effectiveness.csv: per service_level per priority. service_level, priority, backoff_type (from retry config), total_jobs, retry_job_count (jobs with retries), recovery_rate (retried jobs ending OK / retry_job_count), avg_retries (mean retry_count of retried jobs), avg_expected_delay, avg_actual_delay, avg_delay_deviation (all of retried jobs), total_penalties (sum), first_attempt_success_rate (single-attempt jobs that succeeded / single-attempt jobs), exhaustion_rate (EXHAUSTED / total_jobs).
incident_impact.csv: per incident. incident_id, pool_id, region, severity, duration_minutes, capacity_impact_pct, affected_jobs (count), affected_attempts (count), failed_attempts (non-OK during overlap), sla_breaches (affected jobs with breach), penalties_incurred (sum of affected job penalties), affected_tenants (distinct count).
tenant_profile.csv: per tenant. tenant_id, service_level, region, months_active, total_jobs, overall_success_rate, total_cost (sum across months), avg_monthly_cost, total_penalties, total_sla_breaches, avg_retry_rate (mean of monthly retry_rates), final_reliability_score (last month), final_health_status, worst_health_status (priority: CRITICAL > DEGRADED > AT_RISK > STABLE > HEALTHY), consecutive_healthy_months (trailing count from most recent month), longest_healthy_streak (maximum consecutive HEALTHY months across all months), total_incident_jobs, consecutive_breach_months (trailing count of months from most recent where sla_breaches > 0), avg_penalty_per_breach (total_penalties / max(1, total_sla_breaches)).
